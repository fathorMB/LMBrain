# Knowledge Base

The Project Lead maintains concise, evidence-based pages here. Start with:

- `architecture.md` — components, boundaries, and data flows.
- `codebase-map.md` — important directories and ownership.
- `domain.md` — concepts, actors, and business language.
- `setup-and-run.md` — local setup, commands, and verification.
- `integrations.md` — external services, interfaces, and credentials policy.
- `glossary.md` — project-specific terminology.

Use normal Markdown and `[[wikilinks]]`. Avoid copying code details that will rapidly become stale; link to paths and explain intent instead.
